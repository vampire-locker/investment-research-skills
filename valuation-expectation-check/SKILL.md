# 股价预期验证

## 核心定位

这个 skill 用来把已有的公司研究、行业研究、财务分析或护城河判断，映射到当前股价和估值预期上。

不要直接回答“能不能买”“目标价多少”“该不该卖”。应回答：

- 当前价格大致隐含了什么市场预期；
- 这些预期和前面的基本面分析是否匹配；
- 哪些因素已经被市场计入，哪些可能还没有被充分计入；
- 后续哪些指标会推动重估或杀估值；
- 当前风险收益结构更偏向“预期过高”“预期合理但需要验证”，还是“市场可能低估了长期变化”。

## 证据纪律

涉及当前股价、市值、企业价值、估值倍数、盈利预测、利率、汇率、指数估值、行业可比公司或分析师一致预期时，必须先获取最新数据或要求用户提供数据。不要用过时价格做判断。

把结论分为：

- 已确认事实：当前股价、市值、财务数据、估值倍数、公司指引、市场一致预期等可验证信息；
- 基于事实的合理推断：市场可能隐含的增长、利润率、现金流、份额、周期位置或终局假设；
- 需要进一步验证的假设：缺少一致预期、缺少分部利润、周期价格不透明、客户订单不透明等情况。

如果无法获取最新市场数据，明确说明缺口，并给出需要补充的数据清单。

## 工作流程

1. 明确用户是在独立询问估值，还是基于前文公司/行业/护城河分析追问当前股价。
2. 获取或整理最新价格和估值基础数据：股价、市值、净现金/净债务、企业价值、收入、利润、自由现金流、股本、管理层指引和一致预期。
3. 根据公司类型选择合适框架：成长股、周期股、平台型公司、重资产基础设施、金融类公司、资源品公司或亏损成长公司。
4. 默认使用“简版输出结构”，只在用户明确要求完整估值分析、需要正式深度报告、公司估值争议很大、业务类型复杂、需要三情景分析，或需要并入完整研究报告时，才读取 `references/valuation-framework.md` 并使用完整框架。
5. 不给买入/卖出建议；输出市场预期、兑现难度、估值敏感点、重估条件和必须重新评估的信号。

## 默认简版输出结构

除非用户明确要求完整分析或问题明显复杂，默认只输出以下五个中文标题。不要为了显得完整而展开九段框架。

## 一、当前估值基础

简要列出股价日期、市值、主要估值倍数、核心财务数据和数据缺口。无法获取最新数据时，直接说明缺口并列出用户需要补充的数据。

## 二、当前价格隐含的市场预期

用简洁语言反推市场大致计入的增长、利润率、现金流、份额、周期位置或护城河假设。

## 三、与基本面分析的匹配度

对照已有公司研究、行业研究、财务分析或护城河判断，说明当前估值是匹配、偏乐观、偏保守，还是关键假设尚待验证。

## 四、上行验证点与下行风险

分别列出推动重估的 2-4 个关键验证点，以及导致杀估值或重新评估的 2-4 个风险信号。

## 五、结论与跟踪指标

用 3-5 条短句总结：当前价格主要反映什么预期、预期兑现难度、最大上行验证点、最大下行风险、接下来最值得跟踪的指标。最后补一段总判断：当前价格更像是“预期已经充分甚至偏高”“预期合理但需要继续验证”，还是“市场可能低估了长期变化”。

## 完整框架使用条件

只有在以下任一情况成立时，才读取并使用 `references/valuation-framework.md` 的完整九段框架：

- 用户明确要求“完整估值分析”“深度估值分析”“正式估值报告”或类似表述。
- 用户要求做乐观、基准、悲观三情景分析。
- 公司是周期股、资源品、金融股、亏损成长公司、重资产基础设施，或业务分部复杂到简版无法解释清楚。
- 当前估值争议很大，需要系统拆解市场分歧、可比公司和估值敏感性。
- 输出需要并入完整投资研究报告，并且用户要求保留完整估值章节。

## 与其他 skill 的关系

这个 skill 可以独立使用，也可以承接其他投资研究 skill 的结果。典型用法：

- 在 `company-moat-research` 之后追问：“根据前面的分析，如何看待它当前股价？”
- 在财报分析之后追问：“这次业绩后，当前估值反映了什么预期？”
- 在行业研究之后追问：“龙头公司当前股价是否已经计入行业景气？”

如果用户没有提供前文分析，要先快速建立最小基本面上下文，再进行估值预期检查。
