# 一站式股票研究

## 核心定位

这个 skill 把 `company-moat-research`、`valuation-expectation-check` 和 `integrated-equity-research-report` 串联成一条完整流水线。用户只需提供公司名称，skill 会依次完成护城河分析、估值验证和报告沉淀，最终输出一份可归档的 Markdown 研究报告。

不要跳过任何一步，不要合并步骤。每一步的结论是下一步的输入材料。

## 输入

用户只需提供：

- 公司名称（必填），例如"微软""台积电""腾讯"。
- 股票代码或交易所（可选），有则更好，没有也能启动。
- 保存路径（可选），有则保存到指定目录，没有则保存到当前工作目录。

## 工作流程

### 第一步：护城河研究

调用 `company-moat-research` 的完整框架，对目标公司执行护城河分析。必须覆盖：

- 业务边界与证据边界。
- 行业从 0 进入的关键流程、最难突破的环节、需要的资金和资源。
- 公司护城河来源、护城河能否转化为利润和现金流、什么情况下护城河会失效。
- 财务与商业模式验证。
- 已确认事实、合理推断和待验证假设的明确区分。

不要编造数据。如果信息不足，在该步骤内标明缺口，继续执行下一步。

### 第二步：股价预期验证

基于第一步的护城河研究结论，调用 `valuation-expectation-check` 的完整框架，对目标公司当前股价执行估值预期验证。必须覆盖：

- 获取或整理最新股价、市值、企业价值、主要估值倍数、核心财务数据、数据日期和来源。
- 当前价格隐含了什么市场预期。
- 这些预期与第一步护城河研究的基本面判断是否匹配。
- 什么情况下会重估，什么情况下会杀估值。
- 未来 3-5 个核心跟踪指标。

涉及当前股价、市值、估值倍数或一致预期时，必须先获取最新数据或要求用户提供。不要用过时数据做判断。

### 第三步：综合报告生成

将第一步和第二步的全部结论，调用 `integrated-equity-research-report` 的框架，生成一份统一格式的 Markdown 研究报告。必须遵循：

- 使用 `assets/report-template.md` 中的标题和顺序，不要改写标题。
- 合并重复内容。护城河章节展开业务模式和财务质量，估值章节只引用与价格预期直接相关的部分。
- 明确区分已确认事实、合理推断和待验证假设。
- 没有可靠数据的部分标为"待补充"。
- 不给买入、卖出、目标价或仓位建议。
- 不使用 `$` 符号。货币金额用「美元」「港元」「人民币」等文字代替；股票代码直接写纯字母（如 NVDA、AAPL），不加 `$` 前缀；不使用 LaTeX 数学公式语法。这样可以避免报告导入飞书等工具时 `$` 被误解析为公式标记导致内容丢失。
- 使用中文输出，除非用户明确要求英文。

## 输出规则

- 默认保存到当前工作目录，文件名为 `{公司名}-研究报告-{YYYYMMDD}.md`，日期为报告生成当天。
- 如果用户指定了保存路径，保存到用户指定的目录。
- 无论是否保存文件，都在对话中输出完整 Markdown 报告。
- 逐步执行，不要跳步。每一步完成后简要说明进展，再进入下一步。
- 如果某一步因信息不足无法完成，说明缺口，继续执行后续步骤，在最终报告中标为"待补充"。

## 质量检查

完成前检查：

- 是否三步都执行了，没有跳过。
- 护城河结论和估值预期是否连接起来了，而不是并排堆放。
- 是否保留公司名、股票代码、报告日期、数据截止日期和股价日期。
- 是否明确区分事实、推断和假设。
- 是否列出未来 3-5 个核心跟踪指标。
- 是否避免了买入、卖出、目标价和仓位建议。
- 是否所有标题都来自报告模板。
