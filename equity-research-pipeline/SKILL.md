# 一站式股票研究

## 共享规则

本技能遵守仓库根目录 `../RULES.md` 的全部共享规则，包括：输出格式约束、证据分层纪律、禁止买卖建议、默认中文输出、数据时效要求和质量检查清单。本文件不再重述这些规则。

## 核心定位

这个 skill 把 `company-moat-research`、`valuation-expectation-check` 和 `integrated-equity-research-report` 串联成一条完整流水线。用户只需提供公司名称，skill 会依次完成护城河分析、估值验证和报告沉淀，最终输出一份可归档的 Markdown 研究报告。

不要跳过任何一步，不要合并步骤。每一步的结论是下一步的输入材料。

## 输入

用户只需提供：

- 公司名称（必填），例如"微软""台积电""腾讯"。
- 股票代码或交易所（可选），有则更好，没有也能启动。
- 保存路径（可选），有则保存到指定目录，没有则保存到当前工作目录。

## 工作流程

### 第零步：数据基线建立

在开始任何分析之前，先建立本次研究的数据基线。这一步不是独立分析，而是为后续护城河研究和估值验证提供统一的数据输入，避免各步使用不同日期的数据。

**数据基线检查清单：**

| 数据项 | 来源 | 要求 | 缺失处理 |
| --- | --- | --- | --- |
| 公司全称 | 用户输入 | 必填 | 缺失时要求用户提供 |
| 股票代码 | 用户输入或搜索 | 尽量获取 | 可不填 |
| 交易所 | 用户输入或搜索 | 尽量获取 | 可不填 |
| 当前股价和日期 | 搜索最新市场数据 | 必填 | 标注"待补充"，估值步骤将相应降级 |
| 市值 | 搜索最新市场数据 | 必填 | 标注"待补充" |
| 最近一期年报/季报数据（收入、利润、自由现金流） | 搜索最新财报 | 尽量获取 | 标注"待补充"及缺失字段 |
| 一致预期（如有） | 搜索公开一致预期 | 可选 | 没有时直接跳过，不编造 |
| 汇率（如非本币） | 搜索 | 如需换算 | 标注换算日期和来源 |

**执行规则：**
- 所有时效性数据必须标注日期（股价日期、财报截止日期、一致预期日期）。
- 数据基线建立后，在进入第一步前向用户简要汇报数据获取情况：「已获取 [公司名] 当前股价 [价格]（[日期]）、市值 [市值]、最新财报截止 [日期]。以下数据暂缺：[缺失项列表]。将继续执行研究流程，缺失项在报告中标为待补充。」
- 如果核心数据（股价、市值）无法获取，仍继续执行，但在估值步骤中自动降级为定性讨论，不给定量估值判断。
- 数据基线结果传递给后续 `company-moat-research` 和 `valuation-expectation-check`，作为共享输入材料。

### 第一步：护城河研究

调用 `company-moat-research` 的完整框架，对目标公司执行护城河分析。必须覆盖：

- 业务边界与证据边界。
- 行业从 0 进入的关键流程、最难突破的环节、需要的资金和资源。
- 公司护城河来源、护城河能否转化为利润和现金流、什么情况下护城河会失效。
- 财务与商业模式验证。
- 已确认事实、合理推断和待验证假设的明确区分。

不要编造数据。如果信息不足，在该步骤内标明缺口，继续执行下一步。

### 第二步：股价预期验证

基于第一步的护城河研究结论，调用 `valuation-expectation-check` 的完整框架，对目标公司当前股价执行估值预期验证。必须覆盖：

- 获取或整理最新股价、市值、企业价值、主要估值倍数、核心财务数据、数据日期和来源。
- 当前价格隐含了什么市场预期。
- 这些预期与第一步护城河研究的基本面判断是否匹配。
- 什么情况下会重估，什么情况下会杀估值。
- 未来 3-5 个核心跟踪指标。

涉及当前股价、市值、估值倍数或一致预期时，必须先获取最新数据或要求用户提供。不要用过时数据做判断。

### 第三步：综合报告生成

将第一步和第二步的全部结论，调用 `integrated-equity-research-report` 的框架，生成一份统一格式的 Markdown 研究报告。必须遵循：

- 使用 `integrated-equity-research-report` 技能下的 `references/report-template.md` 中的标题和顺序，不要改写标题。
- 合并重复内容。护城河章节展开业务模式和财务质量，估值章节只引用与价格预期直接相关的部分。
- 明确区分已确认事实、合理推断和待验证假设。
- 没有可靠数据的部分标为"待补充"。

## 输出规则

- 默认保存到当前工作目录，文件名为 `{公司名}-研究报告-{YYYYMMDD}.md`，日期为报告生成当天。
- 如果用户指定了保存路径，保存到用户指定的目录。
- 无论是否保存文件，都在对话中输出完整 Markdown 报告。
- 逐步执行，不要跳步。每一步完成后简要说明进展，再进入下一步。
- 如果某一步因信息不足无法完成，说明缺口，继续执行后续步骤，在最终报告中标为"待补充"。

## 质量检查

完成前检查：

- 是否四步都执行了，没有跳过（第零步数据基线、第一步护城河、第二步估值验证、第三步报告生成）。
- 护城河结论和估值预期是否连接起来了，而不是并排堆放。
- 是否保留公司名、股票代码、报告日期、数据截止日期和股价日期。
- 是否明确区分事实、推断和假设。
- 是否列出未来 3-5 个核心跟踪指标。
- 是否避免了买入、卖出、目标价和仓位建议。
- 是否所有标题都来自报告模板。
