# 上下文窗口优化 设计文档

**版本：** v1.0-draft

## 一、背景

### 1.1 reverse-skill 的做法

reverse-skill 的 RULES.md 中有一个明确的设计决策：

> LLM attention is highest in the first and last 10% of any instruction file, decaying in the middle. Critical actions are placed in those high-attention zones. The "MUST NOT skip" items and Checklist are positioned in the last 10%.

### 1.2 当前状态

investment-research-skills 各 SKILL.md 的文件结构从未考虑过 LLM 注意力分布。以 `valuation-expectation-check/SKILL.md`（112 行）为例：

```
开头 10%（1-11 行）：  frontmatter + 标题 + 核心定位 ← 注意力高
中间 80%（12-100 行）：证据纪律、输出约束、工作流、简版结构、完整框架条件 ← 注意力低
末尾 10%（101-112 行）：skill 关系                ← 注意力高，但内容不重要
```

**问题：最需要被遵守的"不给买卖建议"在中间偏前位置，"质量检查"根本没有（在 equity-research-pipeline 才有）。**

---

## 二、优化原则

LLM 对系统提示词的注意力分布大致遵循 U 型曲线：
- **首 10-20%**：注意力最高
- **中间 60-70%**：注意力衰减
- **末 10-20%**：注意力回升

因此文件布局应遵循：

| 位置 | 放置内容 | 理由 |
| --- | --- | --- |
| **开头** | "你是谁" + "最大禁忌" | 建立角色认知和行为边界 |
| **中间** | 工作流步骤、框架引用、输出格式细节 | 执行时自然会参考 |
| **末尾** | "完成前必须逐项检查" + 常见违规 | 作为最后防线 |

## 三、具体优化

### 3.1 共享规则的上下文优化

`RULES.md`（约 100 行）按注意力模式重排：

```
开头 10%：
  一、最大禁止事项（"禁止买卖建议"提至最前，这是最容易犯的错）
  二、核心行为边界（证据不能编造、时效数据必须标日期）

中间 70%：
  三、输出格式约束（$ 禁用、中文、链接规范）
  四、证据纪律（三个层级 + 升级路径）
  五、数据时效要求

末尾 10%：
  六、完成前质量检查清单（逐项打勾）
```

关键改动：把"禁止买卖建议"从中间位置提升到开头（紧接 frontmatter 之后），把质量检查清单放在文件末尾。

### 3.2 子技能 SKILL.md 的上下文优化

以 `company-moat-research`（精简后约 28 行）为例：

```
开头：
  - 遵守 ../RULES.md（引用共享规则）
  - "不要直接评价好坏。假设你是新进入者要挑战它" ← 核心方法论，最高注意力

中间：
  - 三个视角（创业者、产业研究员、长期投资者）
  - 工作流步骤
  - 框架引用和输出格式

末尾：
  - "结论部分先列 5 条短句再给总判断"
  - 回到共享规则的质量检查（引用 RULES.md）
```

28 行的短文件不需要刻意优化，首尾自然覆盖大部分内容。但 **80 行以上的文件**（`valuation-expectation-check`、`equity-comparison-advisor`、`equity-research-pipeline`）需要检查：最关键的禁忌是否在末尾被重申。

### 3.3 长文件的尾部补全

对于 80+ 行的子技能 SKILL.md，在末尾增加：

```markdown
## 完成前提醒

重新确认：不给出买卖建议 / 标注了数据日期和来源 / 区分了事实与推断 / 没有 `$` 符号 / 中文输出。详见 `../RULES.md` 第七条。
```

| 维度 | 评估 |
| --- | --- |
| **价值** | 中低。无法量化测量"AI 遵守规则率"的变化，但可以从 first principles 推断收益 |

| **优先级** | 🟢 低。效果无法验证，属于微优化。建议在共享规则层改造时顺便做（RULES.md 章节排序），不单独立项 |
