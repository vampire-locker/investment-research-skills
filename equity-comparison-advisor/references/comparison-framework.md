
# 多公司股票横向比较框架

## 使用原则

先表格，后解释；先横向比较，后单家公司补充；先风险收益，后操作倾向。

不要让评分替代判断。评分只用于排序和暴露分歧，最终结论必须解释背后的业务、估值和风险原因。

完整报告必须使用固定章节结构。不要新增、删除或改名一级章节，除非用户明确要求简版或局部问答。

固定建议词汇：

- 优先研究
- 可分批配置
- 等待估值消化
- 小仓位观察
- 财报后再评估
- 暂列观察

链接规则：

- “输入报告”“数据来源”“来源文件”等表格列默认写内联代码格式的文件名或路径，例如 `Apple-研究报告-20260622.md`。
- 不要把文件名自动包装成 Markdown 链接。
- 只有在已经验证目标 URL、相对路径或绝对路径真实存在并可访问时，才允许使用 Markdown 链接。
- 禁止输出 `[报告名](报告名)`、`[来源](#)`、`[文件](未知)` 这类无效链接。

## 输出模式与章节

完整报告和归档报告必须在正文标题下、第一章前输出固定报告元信息块：

```markdown
**报告日期：** YYYY-MM-DD
**数据边界：** 研究报告截至 YYYY-MM-DD；股价约截至 YYYY-MM-DD；估值或市场数据参考 YYYY-MM-DD 公开市场数据；缺失项见第二章
**货币单位：** 美元/港元/人民币等；如多币种则逐项说明
**输出模式：** 默认完整报告/归档模式/简版模式/局部问答
```

统一使用“数据边界”，不要使用“数据日期”作为总字段名。无法确认的日期写“待补充”；没有补充最新市场数据时，明确写“未补充最新市场数据，当前估值判断需二次验证”。

### 默认完整报告

严格使用以下章节：

1. 一、执行摘要
2. 二、数据来源与报告边界
3. 三、横向总览表
4. 四、核心维度评分表
5. 五、维度排序表
6. 六、风险收益矩阵
7. 七、投资者类型匹配表
8. 八、买入、等待和重新评估条件表
9. 九、公司逐一点评
10. 十、最终优先级与后续跟踪清单

### 简版模式

用户明确要求简版时，只输出：

1. 一、执行摘要
2. 二、横向总览表
3. 三、风险收益矩阵
4. 四、买入、等待和重新评估条件表
5. 五、结论

### 归档模式

用户要求保存 Markdown 时，使用默认完整报告结构，并写入 Markdown 文件。文件名优先使用用户指定路径；否则使用 `{主题}-横向比较报告-{YYYYMMDD}.md`。

## 信息抽取字段

从每份研究报告中提取以下字段；缺失时写“待补充”。

| 字段 | 说明 |
| --- | --- |
| 公司 | 公司名称和股票代码 |
| 报告日期 | 原始研究报告生成日期 |
| 价格日期 | 报告中股价或估值数据对应日期 |
| 核心业务 | 当前主要收入、利润或价值来源 |
| 护城河来源 | 网络效应、规模效应、切换成本、品牌、生态、专利、供应链、数据、渠道等 |
| 增长驱动 | 未来 3-5 年最关键的增长变量 |
| 估值状态 | 预期偏高、合理待验证、可能低估、数据不足等 |
| 上行验证点 | 支持重估或继续增长的关键指标 |
| 下行风险 | 杀估值、增长失速、监管、竞争、周期、资本开支等 |
| 监管与ESG风险 | 行业监管趋势、反垄断、数据合规、环境政策等关键监管变量；缺失时写"待补充" |
| 跟踪指标 | 后续需要持续观察的数据 |
| 管理层与执行力 | 管理团队能力、资本配置记录、战略执行力评估；缺失时写"待补充" |

## 评分口径

使用 1-5 分，允许半分。无法判断时写“待补充”，不要强行打分。

| 维度 | 5 分含义 | 3 分含义 | 1 分含义 |
| --- | --- | --- | --- |
| 商业质量 | 护城河强、现金流好、利润池稳定、竞争格局清晰、管理层能力与资本配置记录优秀 | 有优势但受周期、竞争或业务转型影响 | 护城河弱、利润质量差或竞争恶化 |
| 增长确定性 | 3-5 年增长驱动清晰且可验证 | 有增长机会但依赖外部周期或执行 | 增长路径模糊或高度不确定 |
| 估值吸引力 | 当前预期保守或估值与质量明显不匹配 | 估值大致合理但需要验证 | 价格已充分透支乐观预期 |
| 风险可控性 | 主要风险慢变量、可跟踪、下行边界较清楚 | 存在重要风险但有验证指标 | 风险非线性、难跟踪或单点依赖严重 |
| 预期差空间 | 市场可能低估长期变化或利润弹性；或市场过度悲观，存在估值修复空间 | 市场定价大致平衡 | 市场已高度一致乐观 |

## 结论写法

最终结论必须包含：

- 第一优先级公司及原因。
- 最需要等待估值或财报验证的公司。
- 基本面很好但当前预期可能过高的公司。
- 风险收益最不清晰、暂不适合直接配置的公司。
- 需要补充的数据和下一次更新触发条件。

建议使用条件化表述：

- “在最新估值没有明显透支的前提下，可作为优先研究对象。”
- “如果未来两个财报周期收入增速和毛利率继续兑现，可提高配置优先级。”
- “如果估值仍停留在高预期区间，而增长指标开始放缓，应转为等待。”

不要使用：

- “一定买入”
- “无脑买”
- “稳赚”
- “目标价必到”
- “适合所有人”
