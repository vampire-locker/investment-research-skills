# 综合股票研究报告

## 共享规则

本技能遵守仓库根目录 `../RULES.md` 的全部共享规则，包括：输出格式约束、证据分层纪律、禁止买卖建议、默认中文输出、数据时效要求和质量检查清单。本文件不再重述这些规则。

## 核心定位

这个 skill 负责把已有投资研究材料整理成一份结构稳定、可归档的 Markdown 报告。

不要把它当成第三套分析框架。它不重新做护城河研究，不重新做估值分析，不主动补充未经验证的数据。它只做合并、去重、压缩、统一标题和证据分层。

## 前置条件检查

在开始整合报告之前，必须先检查输入材料是否满足最低要求。不满足时，不要强行生成报告。

### 最低输入要求

至少需要以下两类材料中的 **一类** 才能生成有效报告：

**A. 护城河研究结论**（来自 `company-moat-research` 或等效材料）：
- 公司业务边界和主要利润来源
- 行业进入壁垒或关键资源判断
- 护城河来源及可持续性评估
- 上述结论的证据等级标注（事实/推断/假设）

**B. 估值基础数据**（来自 `valuation-expectation-check` 或用户提供的等效材料）：
- 当前股价、市值和日期
- 至少一个核心估值倍数（PE、PS、EV/EBITDA 等）
- 最近一期收入和利润数据
- 数据截止日期和来源

### 材料不足时的处理

| 缺失情况 | 处理方式 |
| --- | --- |
| A 和 B 都缺失 | 不生成报告。回复：「当前缺少生成研究报告所需的核心材料。建议先用 `equity-research-pipeline` 一键完成护城河研究和估值验证，或者分别调用 `company-moat-research` 和 `valuation-expectation-check` 生成基础材料后再来整合。」 |
| 只有 A，缺 B | 仍可生成报告，但估值相关章节（六、七、八、九）全部标为「待补充」。在报告开头说明：「本报告缺少估值基础数据，估值章节已跳过。如需补充，请提供当前股价、市值和最新财报数据，或调用 `valuation-expectation-check` 生成估值分析材料。」 |
| 只有 B，缺 A | 仍可生成报告，但护城河相关章节（三、四、五）全部标为「待补充」。在报告开头说明：「本报告缺少护城河研究材料，行业和竞争优势章节已跳过。如需补充，请调用 `company-moat-research` 生成护城河分析材料。」 |
| A 和 B 都有但部分字段缺失 | 正常生成报告，缺失字段在对应位置标为「待补充」，并在第十章汇总所有数据缺口。 |

### 质量红线
- 禁止从公司名称直接推断护城河或估值结论来填充缺失章节。
- 禁止使用未标注日期和来源的价格或财务数据。
- 如果 A 或 B 材料来自当前对话中的分析，直接引用；如果用户只提供了公司名称，必须先说明缺材料，再引导用户补齐。

## 输入材料

优先使用用户已经提供或当前对话中已经生成的材料：

- `company-moat-research` 的护城河研究结果。
- `valuation-expectation-check` 的股价预期验证结果。
- 用户提供的财报、年报、电话会、投资者日、行业资料或笔记。
- 用户提供的当前股价、市值、估值倍数和数据日期。

如果关键材料缺失，不要编造。直接在报告中标为"待补充"或"需要进一步验证"。

## 工作流程

1. 识别目标公司、股票代码、交易所、报告日期、数据截止日期、价格日期和币种；缺失时保留空值或标为"待补充"。
2. 判断用户要的是简版归档还是正式报告。默认生成正式 Markdown 报告；如果用户明确要求简短，只保留核心结论、护城河摘要、估值摘要和跟踪指标。
3. 读取 `references/report-template.md`，使用其中固定标题和顺序生成报告。
4. 从护城河材料中提取业务边界、进入壁垒、关键资源、公司优势、护城河压力测试、商业模式验证和长期跟踪条件。
5. 从估值材料中提取当前估值基础、价格隐含预期、基本面匹配度、上行验证点、下行风险和跟踪指标。
6. 合并重复内容。业务模式和财务质量只在护城河章节展开；估值章节只引用和价格预期直接相关的部分。
7. 对明显冲突的材料，不要强行调和；在"待验证问题与数据缺口"中列出冲突点。
8. 保留报告可读性。不要机械搬运两份原始分析，应该合并同类项、删掉重复段落、保留关键判断。

## 报告质量检查

完成前检查：

- 是否保留公司名、代码、报告日期、数据截止日期和价格日期。
- 是否明确说明哪些是事实、哪些是推断、哪些需要验证。
- 是否把护城河结论和估值预期连接起来，而不是并排堆放两份报告。
- 是否列出未来 3-5 个最值得跟踪的指标。
- 是否避免买入、卖出、目标价和仓位建议。
- 是否所有一级、二级标题都来自模板。

## 保存文件

如果用户要求保存为文件，默认使用 Markdown 文件，并采用清晰、通用的文件名：

`{公司名}-研究报告-{YYYYMMDD}.md`

如果用户提供目录或文件名，优先使用用户指定路径。

## 完成前提醒

提交输出前再次确认：护城河结论和估值预期已连接（非并排堆放）/ 所有标题来自模板 / 不给出买卖建议 / 区分了事实与推断 / 没有 `$` 符号 / 中文输出。详见 `../RULES.md`。
